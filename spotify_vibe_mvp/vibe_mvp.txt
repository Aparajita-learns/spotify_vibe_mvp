7     # Spotify Vibe — AI-Native MVP Implementation Plan 

8      

9     ## 0. Purpose 

10      

11     Build a separate AI-native MVP called **Spotify Vibe**. 

12      

13     This MVP is NOT the review analysis system. 

14      

15     It should simulate a Spotify-like homepage and demonstrate a new AI-native feature: 

16     - ask the user what mood/vibe they are in 

17     - generate ranked song recommendations from best match to least match 

18     - highlight the top choice in a subtle green box 

19     - explain why the top choice is the best fit right now 

20      

21     The MVP must feel product-like, not like an admin dashboard. 

22      

23     --- 

24      

25     ## 1. Product Goal 

26      

27     Demonstrate an AI-native discovery experience that improves meaningful music discovery by: 

28     - collecting real-time mood intent from the user 

29     - producing transparent recommendations 

30     - explaining why a song was chosen 

31     - reducing the “black box” feeling of recommendations 

32      

33     This MVP should be visually inspired by Spotify’s dark UI, but should not require Spotify internal systems.

7. Recommendation Architecture 

7.1 Core Pipeline 

1     User mood input 

2        ↓ 

3     Mood parser 

4        ↓ 

5     Mood profile 

6        ↓ 

7     Candidate generator 

8        ↓ 

9     Scoring engine 

10        ↓ 

11     LLM-assisted reasoning 

12        ↓ 

13     UI render 

7.2 Components 

A. Mood Parser 

Input: 

free text like: 

“I want something calm but not sad” 

“late night city vibes” 

“I need aggressive gym music” 

“I’m nostalgic and slightly emotional” 

Output: 

1     { 

2       "primary_mood": "calm", 

3       "secondary_mood": "hopeful", 

4       "energy_target": 0.35, 

5       "valence_target": 0.62, 

6       "tempo_range": [80, 105], 

7       "context": ["night", "solo"], 

8       "exploration_mode": "safe" 

9     } 

B. Candidate Generator 

Select candidate songs by: 

genre compatibility 

mood tag overlap 

broad numerical fit on energy/valence/tempo 

optional diversity rule 

C. Ranker 

Compute score for each candidate. 

D. Reasoner 

Use LLM to explain why the top song is the best match. 

 

8. Recommendation Logic 

8.1 Deterministic Scoring Formula 

Use weighted scoring: 

1     final_score = 

2         0.30 * mood_tag_overlap + 

3         0.20 * energy_similarity + 

4         0.15 * valence_similarity + 

5         0.10 * tempo_similarity + 

6         0.10 * context_overlap + 

7         0.10 * lyrics_theme_match + 

8         0.05 * popularity_balance 

Notes 

normalize all values to 0–1 

allow experimentation with weights 

keep scoring transparent and reproducible 

8.2 Ranking Rules 

rank candidates by final score descending 

return top 10 

mark rank 1 as top pick 

if scores are too close, optionally ask LLM to break tie using metadata summary 

 

9. LLM Role 

Use LLM for: 

parsing vague mood input into structured mood profile 

generating explanation for the top recommendation 

optional fallback if user input is ambiguous 

Do NOT use LLM as the only recommender. The recommendation pipeline should still work without LLM if needed.
